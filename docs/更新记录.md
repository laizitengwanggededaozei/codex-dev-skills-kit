# 更新记录

本文件记录 `codex-dev-skills-kit` 的通用版本进展、交付范围和后续方向。
不要记录个人账号、本地路径、密钥配置、业务项目细节或一次性操作日志。
专属项目的分析材料应保留在目标项目自己的 `docs/ai-dev/` 记录中。

## 2026-06-01 初始化上下文能力增强

### 本次目标

本次更新聚焦 `setup-codex-dev-workspace` 的初始化质量，目标是让项目或多仓库
工作区在初始化后不仅有目录结构，还能形成可复用的编码前上下文。

重点补齐：

- 初始化 skill 对项目事实探索的要求。
- 结构初始化和编码上下文初始化的状态区分。
- 多仓库工作区与单仓库的初始化口径。
- 子代理只读探索的可选协作方式。
- 探索结果进入长期上下文层的规则。

### 行为变化

初始化 skill 现在应当按两个层级汇报完成状态：

- `Structure Initialization Complete`: 目录、模板和基础文件已存在，但不代表已经能安全编码。
- `Coding Context Initialization Complete`: 已探索并记录项目目的、仓库边界、入口、命令、验证矩阵、风险和未知项，可支撑后续编码前快速回顾。

当运行环境支持子代理，且用户允许或要求时，初始化过程可以使用只读 explorer
agents 加速探索。但主代理仍负责合并事实、处理冲突、标记未知项并写入
`AGENTS.md`、`docs/ai-dev/01_PROJECT_CONTEXT.md`、
`docs/ai-dev/02_WORKFLOW_RULES.md` 或 `docs/ai-dev/60_交接记录/`。

### 已确认边界

- 本次更新只沉淀通用 skill 能力，不写入专属当前业务项目的信息。
- 本地度量分析、临时进度检查和个人操作记录不作为仓库发布内容。
- 子代理探索是可选加速能力，不是初始化完成的硬依赖。
- `AGENTS.md` 不应承载完整探索日志，长期事实应进入分层文档。
